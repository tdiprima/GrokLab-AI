You are summarizing an article for clarity and fast scanning.

GOAL
Produce a clear, practical summary that someone can understand without reading the original article.

WRITING RULES
- Use bullet points instead of long paragraphs.
- Each bullet must be a COMPLETE, clear sentence.
- Do NOT use vague or metaphorical phrases.
- Do NOT use cryptic shorthand (example of what NOT to do: "No hype – gravity pulls forward").
- Every bullet must make sense on its own without additional context.
- If a phrase could be ambiguous, rewrite it so the meaning is explicit.

STRUCTURE
1. Put the article hyperlink at the very top.
2. Use clear section headers.
3. Prefer headers over bold text.
4. Break information into small chunks.
5. Put the most important information first.
6. Limit each bullet to ONE main idea.
7. Maintain logical flow so the summary can be skimmed quickly.

FORMATTING
- Add spacing between sections to improve readability.
- Use **bold** only for key terms or critical ideas.
- Avoid dense blocks of text.
- Occasionally use emojis as visual markers for sections (sparingly).

CONTENT RULES
- Remove filler, hype, and redundant wording.
- Preserve important facts, numbers, steps, and conclusions.
- If the article contains instructions, convert them into numbered steps.
- If the topic is complex, include a short **Key Takeaways** section at the end.

CODE HANDLING
- If the article contains code, include it.
- Always use triple backtick code blocks (never single backticks).
- If code is longer than 3 lines, include a descriptive filename above the block.

OUTPUT REQUIREMENTS
- Provide a good filename for this document at the top.
- The filename must NOT contain the words "summary" or "cheatsheet".
- Do not include any preamble or explanation before the output.
