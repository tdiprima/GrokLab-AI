You will summarize an article using a two-step process.

The goal is to produce a clear, structured breakdown that preserves the meaning of the article and avoids vague or cryptic bullet points.

----------------------
STEP 1 — INTERNAL ANALYSIS
----------------------

Read the article and internally determine:

* The main thesis
* The major sections or themes
* The most important facts or arguments
* Any processes, frameworks, or instructions
* Key numbers, examples, or evidence

Do NOT output this analysis.

Use it only to ensure the summary is accurate and clear.

----------------------
STEP 2 — STRUCTURED BREAKDOWN
----------------------

Now produce the formatted result.

Place the hyperlink at the very top.

# Filename
Provide a good filename for this response.
Do NOT use the words "summary" or "cheatsheet".

# Core Idea
2–3 bullet points explaining the central idea of the article.

# Key Points
6–12 bullet points covering the most important insights.

# Important Details
Supporting facts, examples, numbers, or clarifications.

# How It Works / Process
If the article explains a system or steps, convert it into a numbered list.

# Key Takeaways
3–5 bullets capturing the most important lessons.

----------------------
WRITING RULES
----------------------

* Each bullet must be a complete sentence.
* Each bullet must express ONE clear idea.
* Avoid vague, metaphorical, or cryptic phrasing.
* Avoid compressed shorthand.
* Every bullet must make sense on its own.
* If a phrase could be ambiguous, rewrite it in plain language.

Example of what NOT to write:
"Growth accelerated — gravity pulled users in."

Correct style:
"User growth increased mainly through organic adoption rather than paid marketing."

----------------------
FORMATTING RULES
----------------------

* Use bullet points instead of paragraphs.
* Use clear section headers.
* Prefer headers over bold text.
* Add spacing between sections.
* Highlight key terms with **bold** when helpful.
* Avoid dense walls of text.
* Occasionally use simple emojis for section markers (sparingly).

----------------------
CODE HANDLING
----------------------

If the article contains code:

* Include the code exactly.
* Use triple-backtick markdown blocks only.
* Never use single backticks for blocks.
* If code is longer than 3 lines, add a descriptive filename above the code block.

----------------------
OUTPUT RULE
----------------------

Do not include any preamble or explanation.

Only produce the formatted result.
